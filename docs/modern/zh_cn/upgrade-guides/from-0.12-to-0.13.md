# 0.12 → 0.13 升级指南

## 警告 & 已知问题

**请务必创建一个新的 TFG 实例，并将你的世界存档复制到新实例中，切勿直接在原实例原地升级！** 这样做更加安全，还能让你预览升级后的世界，方便你回退并做好更充分的准备。

如果你是从 0.11 版本升级，请先查阅 [0.12 升级指南](/modern/zh_cn/upgrade-guides/from-0.11-to-0.12)。

0.12 版本的世界可以安全升级至 0.13，但有一些重要事项需要注意：

# 主世界地形生成

由于服务器端存在一些问题，我们已将 0.12 之前世界的默认地形生成改为**新世界生成**。

- 如果你的世界是用 0.12 创建的，则无需进行任何操作。
- 如果你的世界是在 0.12 之前创建的，并且已经将其更改为使用新世界生成，则无需进行任何操作。
- 如果你的世界是在 0.12 之前创建的，并且想保留旧世界生成，则需要将 `defaultconfigs/tfg-server.toml` 中的 `worldgenOverrides` 设置更改为 `["minecraft:overworld=0"]`。

你无需进行任何操作来生成新的矿井结构，只需探索新的区块即可。

在未来（可能在几个月后），我们可能会停止对旧世界生成的支持，因为同时维护两者的工作量变得很大。

# 幽冥之地翻新

我们已彻底重做幽冥之地。 它现在明显更高了，因此需要清除该维度，以免被传送到基岩之上。

操作步骤：首先备份存档；接着确保已离开幽冥之地，并带走所有想保留的物品；然后保存并退出游戏。 之后进入 Minecraft 实例文件夹，依次打开 `saves`，再进入以存档名命名的文件夹。 删除名为 `DIM-1` 的文件夹。 最后正常启动游戏。 服务器上的操作方式相同。

⚠ ⚠ ⚠ **执行此操作前请务必备份存档** ⚠ ⚠ ⚠

若你曾在幽冥之地打开过 HV 或 LuV 探矿仪的**流体矿脉模式**，或使用过**流体钻机**，还需删除 `.minecraft/saves/(世界名)/data/gtceu_bedrock_fluid.dat` 文件，因为流体矿脉也已发生变化。

# 食物重做

TFC Gourmet 已被移除。若你还有该模组的食物或流体，请在升级前消耗完毕，升级后它们将全部消失。 若你还有该模组的食物或流体，请在升级前消耗完毕，升级后它们将全部消失。

我们将继续以自有内置物品的形式添加新食物（包括部分原先由 TFC Gourmet 添加的内容），以便完全掌控其实现与平衡。

# 流体物流

_机械动力：工厂物流_（Create: Factory Logistics）已被替换为 _机械动力：流体物流_（Create Fluid Logistics），这是机械动力物流系统的另一款流体附属模组，Bug 更少。

你世界中的装罐机应会自动替换为新的流体打包机，流体仪表应会自动替换为普通工厂仪表（现在它们自身即可处理流体）。**如果你大量使用这些设备，更新前请务必再三确认已做好备份！** **如果你大量使用这些设备，更新前请务必再三确认已做好备份！**

# 光影

更新后，若幽冥之地整体呈现红/橙色调，请前往光影设置 > 大气 > 下界维度 > 颜色模式，将其改为「基于群系」，_不&#x8981;_&#x9009;「基于群系（微调）」。 或者，若显示为「自定义」预设，可换一个其他预设。

# 矿石

硅岩已从幽冥之地移除。 它被暂时转移至紫金线性加速器，直至未来在某颗行星上找到合适归宿。

铬的获取方式已完全改变。 由于引入了宝石浆液，红宝石意外成为了铝和铬的最佳来源。 为解决这一问题：

- 红宝石现在是明显更差的铝来源，但仍是良好的铬来源。 同时它也是电路板氧化铝的来源。
- 石榴石砂获得大幅加强，拥有独立处理线，涉及黄色石榴石和钙铬榴石。 你可以在主世界的沙地找到它，或在幽冥之地底层找到大量储量。
- 铬铁矿可通过部分矿石处理获得，也可在月球上找到铬铁矿与黄色石榴石矿石。

随着进程推进，你会解锁处理这三者的更简单方法。

# ComputerCraft

我们新增了一个模组，为 ComputerCraft 启用 UTF-8 支持。

现有 ComputerCraft 程序应无需迁移即可继续运行。 已包含有效 UTF-8 文本的程序在安装该模组后应能正确显示。 此前以乱码保存的程序需要手动修正文本；仅打开并重新保存文件无法自动恢复原始字符。
